## Состав обучающего набора

Из `4 442` принятых учебных записей:

| Отношение к ЖКХ | Количество | Доля |
| --- | ---: | ---: |
| Не относится к ЖКХ | 4 073 | 91.69% |
| Относится к ЖКХ | 351 | 7.90% |
| Не уверен(а) | 18 | 0.41% |

Для дипломной модели это существенный дисбаланс: текущий проверенный датасет
уже полезен для общего классификатора, но положительный класс ЖКХ остается
малочисленным. При дальнейшем сборе данных нужно целенаправленно увеличивать
долю обращений по водоснабжению, отоплению, уборке, дворам, работе УК и
муниципальному благоустройству.

Срез был сформирован командой:

```bash
python manage.py export_annotation_statistics "data/exports/statistics_$(date +%F_%H-%M)"
```

Команда создала:

- `annotation_statistics.json` для точного машинного учета;
- `annotation_statistics.md` для текста диплома;
- `annotation_statistics_infographic.svg` для иллюстрации;
- `participants.csv`, `reviewers.csv` и `approved_taxonomy.csv` для анализа.

Полные CSV участников и инфографика с рейтингом хранятся в локальных
экспортных артефактах, а не в git, поскольку содержат идентификаторы
участников.

## Визуальные отчетные материалы

На основе официального JSON-среза локально сформированы готовые материалы для
доклада и приложений:

- `data/exports/statistics_2026-05-25_06-33/annotation_statistics_infographic.png` -
  компактная инфографика с итогами и топ-10;
- `data/exports/statistics_2026-05-25_06-33/annotation_dashboard_full.svg` -
  расширенный векторный дашборд;
- `data/exports/statistics_2026-05-25_06-33/annotation_dashboard_full.png` -
  расширенный PNG-дашборд с итогами проверки, баллами, проверяющими, топ-10 и
  распределением обучающего набора по всем полям разметки.

Для повторной сборки расширенного SVG из любого последующего среза добавлена
команда:

```bash
python manage.py render_statistics_dashboard \
  data/exports/<statistics-dir>/annotation_statistics.json \
  data/exports/<statistics-dir>/annotation_dashboard_full.svg
```

PNG является локальным отчетным артефактом и намеренно не публикуется в git,
как и CSV со списком участников.
