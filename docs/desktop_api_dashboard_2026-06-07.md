# Desktop-приложение и API мониторинга ОМСУ

Дата фиксации: `2026-06-07`.

Текущая версия desktop-клиента: `0.5.3`.

## Назначение

Добавлен первый рабочий контур для будущего приложения мониторинга ЖКХ и оценки деятельности ОМСУ:

1. Django отдает универсальный JSON API для desktop- и Android-клиентов.
2. API хранит только агрегированный срез по районам/ОМСУ, а не весь поток спарсенных комментариев.
3. PyQt6-приложение строит интерактивную карту, показывает карточки статистики и обновляет ленту последнего комментария.

## Реализованные backend-компоненты

Модели:

| Модель | Назначение |
| --- | --- |
| `OmsuArea` | справочник районов/ОМСУ, базовая информация, руководители, геометрия |
| `OmsuDashboardSnapshot` | текущий агрегированный срез: оценка ОМСУ, прошлый score, счетчики, графики |
| `OmsuLatestComment` | короткий срез последнего комментария для нижней ленты |

API:

| Endpoint | Назначение |
| --- | --- |
| `GET /api/v1/omsu/manifest/` | версия API, интервалы обновления, список endpoint-ов |
| `GET /api/v1/omsu/snapshot/` | общий срез по всем районам/ОМСУ |
| `GET /api/v1/omsu/areas/<slug>/` | подробный срез по выбранному району/ОМСУ |
| `GET /api/v1/omsu/latest-comment/?area=<slug>` | последний комментарий для выбранной территории |

Доступ к API поддерживает ключ:

| Механизм | Заголовок |
| --- | --- |
| основной вариант | `X-OMSU-API-Key: <ключ>` |
| совместимый вариант | `Authorization: Bearer <ключ>` |

Настройки:

| Настройка | Значение по умолчанию | Назначение |
| --- | ---: | --- |
| `OMSU_API_SNAPSHOT_REFRESH_SECONDS` | `300` | период обновления основного среза |
| `OMSU_API_COMMENT_REFRESH_SECONDS` | `2` | период обновления последнего комментария |
| `OMSU_API_REQUIRE_KEY` | `false`, если ключи не заданы | требовать API-ключ |
| `OMSU_API_KEYS` | пусто | список разрешенных ключей через запятую |

Для production-сервера нужно добавить в `.env`:

```env
OMSU_API_REQUIRE_KEY=1
OMSU_API_KEYS=секретный_ключ_для_desktop,секретный_ключ_для_android
```

Команда демо-заполнения:

```powershell
.\.venv\Scripts\python.exe manage.py seed_omsu_demo_data
```

Она создает 8 демонстрационных территорий Нижегородской области с оценками, полигонами, графиками и последними комментариями.

## Реализованные desktop-компоненты

Файлы:

| Файл | Назначение |
| --- | --- |
| `desktop_dashboard/api_client.py` | HTTP-клиент API и fallback-демо при недоступном сервере |
| `desktop_dashboard/main.py` | PyQt6-интерфейс |
| `desktop_dashboard/assets/static_dashboard_snapshot.json` | локальный статический снимок для запуска без сервера |
| `desktop_dashboard/README.md` | команды запуска |
| `desktop_requirements.txt` | desktop-зависимости |

Функции интерфейса:

- карта Нижегородской области с цветом района по `omsu_score`;
- красно-зеленая шкала от `-100` до `+100`;
- черный контур каждого района;
- hover-поднятие района над картой;
- tooltip с оценкой, прошлой оценкой, числом комментариев и последним сообщением;
- клик по району приближает карту к выбранной территории;
- левая информационная панель района;
- центральные графики статистики;
- желтая нижняя лента последнего комментария;
- верхняя шапка раскрывается при наведении и хранит запасные графики;
- drag/drop графиков между шапкой и главным экраном.

## Версия 0.5.0: статическая витрина и стратегический стиль карты

В версии `0.5.0` desktop-клиент временно переведен в локальный режим: сервер
сейчас не обязателен, а базовый снимок берётся из
`desktop_dashboard/assets/static_dashboard_snapshot.json`. Геометрия России,
Нижегородской области и 52 объектов мониторинга остаётся локальной и реальной,
а числовые показатели для объектов строятся как детерминированный демонстрационный
срез до восстановления API/ML-агрегатора.

Карта также получила новый визуальный слой: тонкие контуры, тёмную
картографическую подложку, градиентную заливку территорий, лёгкий объём и
единые uppercase-подписи внутри самих районов/ОМСУ. Малые кликабельные маркеры
сохранены как страховка для трудных участков, но приглушены, чтобы не ломать
стиль карты.

## Версия 0.5.1: камера, подписи и веса статических графиков

В версии `0.5.1` длинные подписи территорий переносятся на две строки и
уменьшаются по доступной геометрии. При выборе района/ОМСУ карта не оставляет
жёлтую точку: выбранный полигон поднимается, получает подсветку, а камера
плавно центрируется на нём.

В региональном режиме карта поддерживает ручное перетаскивание мышью и
масштабирование колесом. Камера ограничена границами сцены, поэтому карту нельзя
увести в пустое пространство.

Статический заменитель API получил более контрастные веса: графики показывают
явно доминирующие категории и заметно меньшие второстепенные значения, чтобы
витрина читалась как аналитический мониторинг, а не как ровный демо-набор.

## Версия 0.5.2: исправление hover и оптимизация карты

В версии `0.5.2` подписи районов вынесены в управляемый слой. Выбранная подпись
поднимается выше выбранного полигона и не пропадает при наведении. Когда выбран
конкретный район/ОМСУ, остальные подписи временно скрываются до возврата к
полной карте области.

Кнопка назад стала двухступенчатой: из выбранного района она показывает
`← Область`, а на полной карте области — `← Россия`.

Для снижения лагов убраны runtime blur/shadow effects с полигонов, включён cache
mode для основных элементов сцены и более дешёвый режим обновления
`QGraphicsView`. Во время drag/zoom карта временно скрывает подписи и отключает
тяжёлый антиалиасинг, затем автоматически возвращает качество.

## Версия 0.5.3: high-res подписи и плавный возврат к области

В версии `0.5.3` полный слой подписей карты области запекается в отдельный
high-resolution pixmap поверх карты. Это снижает нагрузку на сцену и убирает
размытие, вызванное кэшированием отдельных текстовых элементов.

Когда выбран конкретный район, общий baked-label слой скрывается, а подпись
выбранного района остаётся отдельным векторным текстовым объектом без cache
mode. Выбранному полигону возвращены визуальный подъём и тень, но эффект
применяется только к выбранному району, чтобы не повторять старую просадку FPS.

Возврат по кнопке `← Область` теперь плавно анимирует камеру обратно к полной
карте всех районов.

## UI-уточнение после первого запуска

После первого запуска desktop-клиента на production API интерфейс был
подчищен как рабочий мониторинг, а не как сырая демонстрационная форма:

- карта больше не показывает горизонтальные/вертикальные скроллы и не
  приближает выбранный район так, что заголовок и область уезжают за край;
- выбранная территория теперь выделяется утолщенным контуром, а вся область
  остается в кадре;
- карте задана минимальная ширина, чтобы правые графики не схлопывали ее до
  узкой полосы;
- верхняя панель запасных графиков стала компактным hover-drawer: скрытые
  карточки не торчат из свернутой шапки;
- основные карточки графиков получили большие внутренние отступы, а подписи
  категорий теперь обрезаются через ellipsis вместо выхода за границы;
- лишняя системная status bar строка под желтой лентой убрана.

Это не заменяет следующий production-шаг: настоящие геометрии районов/ОМСУ и
реальный агрегатор оценок все еще должны прийти вместо демо-полигонов и
демо-срезов.

## Версия 0.2.0: карта как основной экран

После уточнения исходного пользовательского запроса desktop-клиент переведен
из компоновки "карта слева, дашборд справа" в map-first UX:

- при первом входе рабочее пространство занимает карта России;
- пользователь выбирает регион на карте России;
- камера плавно приближает выбранный регион;
- после входа в регион карта ОМСУ остается основным слоем;
- карточка территории, графики и желтая лента последнего комментария лежат
  поверх карты как overlay;
- кнопка `← Россия` возвращает на общую карту страны;
  версия хранится в `desktop_dashboard/VERSION`.

Ограничение `0.2.0`: карта России и региональная карта пока являются
упрощенной демонстрационной геометрией. Для следующей версии нужно подключить
реальные границы субъектов РФ и ОМСУ.

## Версия 0.3.0: настоящая карта России и slide-панели

В версии `0.3.0` стартовый слой карты России заменен с ручной схемы на
GeoJSON-геометрию:

| Asset | Назначение |
| --- | --- |
| `desktop_dashboard/assets/geodata/rus_adm0_simplified.geojson` | граница РФ уровня ADM0 |
| `desktop_dashboard/assets/geodata/rus_adm1_simplified.geojson` | субъекты РФ уровня ADM1 |
| `desktop_dashboard/assets/geodata/metadata.json` | источник и ссылки скачивания |

Источник геоданных: `geoBoundaries gbOpen`, уровни `RUS/ADM0` и `RUS/ADM1`.
На стартовой карте Нижегородская область определяется по `shapeISO=RU-NIZ`.

Также добавлена анимация выдвижения:

- при входе в регион левая информационная панель выезжает слева;
- правая панель графиков выезжает справа;
- та же механика используется при выборе ОМСУ внутри региона;
- старая ручная карта России осталась только как fallback на случай отсутствия
  GeoJSON-asset.

Ограничение `0.3.0`: внутренние полигоны ОМСУ/районов Нижегородской области
пока остаются демонстрационными. Следующий шаг — заменить их реальными
муниципальными границами и связать с production-агрегатором.

## Версия 0.4.0: реальные объекты мониторинга Нижегородской области

В версии `0.4.0` региональная карта больше не использует условные полигоны.
Desktop-клиент строит карту Нижегородской области по локальному ADM2-слою и
накладывает на него объекты мониторинга из пользовательского списка VK-групп.

| Asset | Назначение |
| --- | --- |
| `desktop_dashboard/assets/geodata/rus_adm2_simplified.geojson` | полный слой `geoBoundaries gbOpen/RUS/ADM2` |
| `desktop_dashboard/assets/geodata/nizhny_monitored_adm2.geojson` | 52 выбранных муниципальных объекта Нижегородской области |
| `desktop_dashboard/assets/monitoring_groups_121121.txt` | локальная копия списка из `C:\Users\OldSkull\Desktop\121121.txt` |
| `desktop_dashboard/assets/monitoring_groups.json` | нормализованная связка `slug -> VK-группы` |

Проверенное сопоставление:

- `240` VK-групп из файла `121121`;
- `52` объекта мониторинга;
- `52` объекта имеют геометрию;
- нет объектов из списка мониторинга без геометрии;
- нет геометрических объектов без записи в списке мониторинга.

UX-изменение: после клика по Нижегородской области сначала показывается вся
карта выбранных объектов мониторинга. Левая информационная панель и правая
панель графиков выезжают только после клика по конкретному району/ОМСУ, поэтому
мелкие территории не закрываются overlay-панелями до выбора.
