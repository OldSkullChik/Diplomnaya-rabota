# Каскадная система разметки и оценки ОМСУ, 2026-06-06

## Цель

Проверить схему, в которой система не заменяет таксономию одной оценкой, а работает каскадом:

1. определяет все 8 осей разметки;
2. применяет логические правила согласованности;
3. отдельно определяет вероятность негативного сигнала по ОМСУ;
4. рассчитывает числовую оценку `omsu_score` и confidence-поля для рейтинга.

## Логические правила согласованности

Включено простое правило:

```text
если jkh_relevance = no,
то jkh_topic = not_jkh,
authority_aspect = not_applicable,
responsible_party = not_applicable
```

Это не меняет смысл текста, а убирает нелогичные комбинации, когда запись признана не-ЖКХ, но зависимые поля остаются про ЖКХ/ответственных.

Эффект на test:

- mean taxonomy macro-F1 вырос с `0.3003` до `0.3032`;
- strict exact match по всем 8 осям вырос с `0.0733` до `0.0885`;
- postprocess затронул `36` test-записей из `655`.

Прирост небольшой, но стабильный и логически безопасный.

## Метрики ОМСУ-слоя

Test split:

| Mode | Coverage | Accuracy | Macro-F1 | Weighted-F1 | Negative F1 |
| --- | ---: | ---: | ---: | ---: | ---: |
| argmax | `1.0000` | `0.8901` | `0.7321` | `0.9083` | `0.5263` |
| threshold `0.69` | `1.0000` | `0.9237` | `0.7635` | `0.9302` | `0.5690` |
| selective | `0.7496` | `0.9939` | `0.9102` | `0.9937` | `0.8235` |

Selective policy:

```text
negative_omsu, если P(negative_omsu) >= 0.85
not_negative_omsu, если P(negative_omsu) <= 0.15
low_confidence во всех остальных случаях
```

Это наиболее надежный режим для автоматического рейтинга ОМСУ:

- около `75%` записей получают уверенное решение;
- сомнительные записи сохраняются как `low_confidence`;
- уверенная часть дает macro-F1 `0.9102`.

## Числовая оценка

`omsu_score` считается по финальным предсказанным осям после consistency-rules.

Test:

- средняя абсолютная ошибка: `13.14` балла;
- в пределах `10` баллов: `0.7344`;
- в пределах `20` баллов: `0.7786`.

Для рейтинга рекомендуется использовать:

- `omsu_score` как численное значение;
- `omsu_decision` как категориальный итог;
- `omsu_negative_probability` как уверенность;
- `omsu_rating_weight = 1.0` только для уверенных решений;
- `omsu_rating_weight = 0.0` для `low_confidence`, если нужен строгий автоматический рейтинг.

## Вывод

Каскадная схема подтвердилась:

- 8 осей остаются обязательными для статистики, фильтров и инфографики;
- логическая постобработка немного улучшает согласованность таксономии;
- отдельный ОМСУ-слой дает качественный рейтинговый сигнал на уверенной части данных;
- главный bottleneck системы - не ОМСУ-рейтинг, а редкие классы в 8-осевой таксономии.

Текущий практический вариант для приложения:

1. прогнать текст через 8-осевую модель;
2. применить consistency-rules;
3. посчитать `omsu_score`;
4. прогнать ОМСУ binary checkpoint;
5. если решение уверенное, учитывать запись в автоматическом рейтинге;
6. если `low_confidence`, хранить запись и показывать в аналитике, но не давать ей полный вес в итоговой оценке ОМСУ.
